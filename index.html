Brigadehub

We're striving to build an out-of-the-box experience for brigade leadership, centered around an easy-to-use admin platform that anyone, with any technological background, can easily use.

Still in pre-alpha, brigadehub is not yet ready for prime-time, but the ideas and tools that are going into it can still be used. And, if your brigade wants to help with development, that option available as well.

Microservices

These are small, granular applications that do one thing, and do that one thing well.
You can install the library as a whole, or just what your brigade needs.

  • slackin

    rauchg - node.js

    A little server that enables public access to a Slack server. Like Freenode, but on Slack. Provides a landing page you can point users to fill in their emails and receive an invite, as well as embed badges.

  • github-add

    sfbrigade - node.js

    A lightweight node application that automatically adds users to the github org. Like slackin, but for github.

  • cfa-slack-checkin

    sfbrigade - node.js

    A microservice to allow hacknight checkins via slash comand. Also provides stats export via CSV.

  • github-auto-label

    sfbrigade - node.js

    A webhook service to auto-label new issues or PRs on submission. Cut and dry.

  • c4sf.me

    sfbrigade - node.js

    A url shortening service for Code for San Francisco. Easily redeployed to other brigades via Heroku.

  • glossary-bot

    codeforamerica - python

    Glossary Bot is a Slack bot that maintains a glossary of terms created by its users, and responds to requests with definitions.
